# CLAUDE.md — Austravel_OSM

## Purpose

Self-hosted OpenStreetMap tile server for the Austravel Safety Net HF radio operations platform. Generates and serves Australian vector tiles from OpenStreetMap data, replacing reliance on third-party tile services.

## Architecture

```
OSM PBF extract  ──►  tilemaker  ──►  PMTiles file  ──►  Martin tile server  ──►  MapLibre GL JS
(Geofabrik)            (Docker)        (static file)      (Docker, port 3000)      (browser)
```

| Component | Role |
|---|---|
| **tilemaker** | Converts OSM PBF → PMTiles vector tiles (Docker: `ghcr.io/systemed/tilemaker:master`) |
| **Martin** | Rust-based tile server serving PMTiles + fonts via HTTP (Docker: `ghcr.io/maplibre/martin:latest`, port 3000) |
| **MapLibre GL JS** | Client-side vector tile renderer (v4.7.1, loaded from CDN in DjangoGO) |
| **PMTiles** | Single-file vector tile archive format, served via HTTP range requests |

## Quick Start

```bash
# 1. Download Australian OSM data
curl -O https://download.geofabrik.de/australia-oceania/australia-latest.osm.pbf

# 2. Download coastline shapefile
curl -O https://osmdata.openstreetmap.de/download/water-polygons-split-4326.zip
mkdir -p coastline && cd coastline && unzip ../water-polygons-split-4326.zip --strip-components=1 && cd ..

# 3. Extract park polygons
pip install osmium shapely fiona pyproj
python extract-parks.py australia-latest.osm.pbf parks/parks.shp

# 4. Generate vector tiles (~20 minutes)
docker run --rm -v "$(pwd)":/data ghcr.io/systemed/tilemaker:master \
    --input /data/australia-latest.osm.pbf \
    --output /data/australia.pmtiles \
    --config /data/config-openmaptiles.json \
    --process /data/process-openmaptiles.lua \
    --store /tmp/tilemaker-store \
    --verbose

# 5. Start Martin tile server
docker compose up -d

# 6. Verify
curl http://localhost:3000/catalog    # Should list 'australia' source
```

## Tile Generation Details

### Tilemaker Configuration (`config-openmaptiles.json`)

Defines vector tile layers with zoom ranges:

| Layer | Min Zoom | Source |
|---|---|---|
| `ocean` | 0 | Coastline shapefile |
| `place` | 0 | OSM PBF |
| `boundary` | 0 | OSM PBF |
| `landcover` | 0 | OSM PBF |
| `park_major` | 2 | Parks shapefile |
| `transportation` | 4 | OSM PBF |
| `park` | 4 | OSM PBF (Lua) |
| `landuse` | 4 | OSM PBF |
| `water` | 6 | OSM PBF |
| `waterway` | 8 | OSM PBF |
| `building` | 13 | OSM PBF |

### Lua Processing (`process-openmaptiles.lua`)

Based on the OpenMapTiles schema. Key customisations:
- **Park handling**: Processes `boundary=national_park`, `boundary=protected_area` (with `protect_class`), and `leisure=nature_reserve`
- **Relation tags**: Reads parent relation tags via `NextRelation()`/`FindInRelation()` for parks stored as boundary relations
- **Names**: Written to `name:latin` attribute (not `name`)
- **Area-based zoom**: `SetMinZoomByArea()` assigns features to lower zoom levels based on polygon area

### Park Extraction (`extract-parks.py`)

Many Australian national parks (e.g., Kakadu, Litchfield) are `type=boundary` relations in OSM, which tilemaker cannot assemble into proper polygons. This script:
1. Scans the PBF for park/protected area relations
2. Collects member way geometries
3. Stitches boundary segments into complete polygons using Shapely
4. Outputs a shapefile used as the `park_major` tile layer source

## MapLibre Style (`style/style.json`)

~60 layers organised as:
- **Base**: background, ocean, landcover (wood, grass, farmland, sand, wetland, rock, ice)
- **Land use**: residential, commercial, military, hospital, school, cemetery
- **Parks**: `park_major` (shapefile, from z2) + `park` (Lua, from z4) with fills, outlines, labels
- **Water**: fills, waterways
- **Roads**: track, path, service, minor, tertiary, secondary, primary, trunk, motorway (with casings)
- **Rail**: dashed lines with hatching
- **Labels**: roads, water, parks, aerodromes, POIs, places (neighbourhood → country)

Colour scheme: light cartographic base (#f2efe9 land, #aad3df water, green parks).

## Map Style Editor (`editor/editor.html`)

Django template providing a Mapbox-Studio-inspired interactive style editor:
- Split-panel: scrollable layer sidebar + full MapLibre map
- Live editing of colours, opacity, line widths, zoom ranges
- Layers grouped into 15 categories
- Export modified style as JSON download
- Integrated into DjangoGO at `/map/`

## Integration with DjangoGO

DjangoGO's `docker-compose.yml` includes the Martin tile server:
```yaml
tileserver:
  image: ghcr.io/maplibre/martin:latest
  ports:
    - "3000:3000"
  volumes:
    - ./tiles:/tiles
  command: ["--font", "/tiles/fonts", "/tiles/australia.pmtiles"]
```

The style JSON is served from DjangoGO's `static/mapstyle/style.json` and references `http://localhost:3000/australia` for tiles and `http://localhost:3000/font/{fontstack}/{range}` for glyphs.

## Key Design Decisions

- **PMTiles over MBTiles**: Single-file format with HTTP range request support — no database needed
- **Martin over tileserver-gl**: Rust-based, lower memory, serves fonts natively from TTF
- **Shapefile park extraction**: Workaround for tilemaker's inability to assemble `type=boundary` relation polygons
- **`name:latin` attribute**: Tilemaker writes names to `name:latin`; MapLibre style uses `["coalesce", ["get", "name:latin"], ["get", "name"]]`
- **Overzooming**: Vector source `maxzoom: 14` allows MapLibre to render z14 tiles at higher zoom levels
- **Self-hosted fonts**: Open Sans Regular/Bold TTF files served by Martin — no external CDN dependencies at runtime
